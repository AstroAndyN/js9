Move circle catalog:

Catalogs are arrays of JavaScript objects having the following elements (as well others that you wish to include):
  • x: image x coordinate
  • y: image y coordinate
  • shape: "circle", "box", "ellipse", "polyon"
  • color: color name or hex number
In addition, for each shape, the appropriate size and position parameters can be specified. These are the same as for regions:
  • circle: radius
  • box: width, height
  • ellipse: eradius.x, eradius.y
  • polygon: points
Catalogs are added to an image by first creating a new shape layer and then adding shapes: 
    JS9.NewShapeLayer(catalog_name, opts);
    JS9.AddShapes(catalog_name, catalog_array, opts);
where catalog_name is an indentifying name for the catalog. and opts is a JavaScript object which can specify default values for any of the parameters above (except for the x and y coordinate). When all else fails, default values are taken from the JS9.Catalog.opts object in JS9.

By default, catalogs are not interactive, but you can make them so in one of two ways:

The first way is to supply one or more event callbacks as properties to the opts object supplied to the JS9.NewShapeLayer() call:

  • onmousedown: function(im, xreg, evt);
  • onmouseup: function(im, xreg, evt);
  • onmousemove: function(im, xreg, evt);
  • onmouseover: function(im, xreg, evt);
  • onmouseout: function(im, xreg, evt);
Each of these functions will be called with the following arguments:
  • im: the image handle for the currently displayed image
  • xreg: the shape object, as described in JS9.GetShapes()
  • evt: the original event object
For example, to define mouseover and mousedown callbacks: 
    opts.onmouseover = function(im, xreg, evt){
        console.log("mouseover: %s %s", im.id, xreg.data.tag);
    };
    opts.onmousedown = function(im, xreg, evt){
        console.log("mousedown: %s %s", im.id, xreg.data.tag);
    };

The second way to make a shape layer interactive is to specify that a tooltip be displayed when hovering over objects in this shape layer. This is done by assigning a tooltip format string to the tooltip property of the opts object supplied to the JS9.NewShapeLayer() call. This string can contain HTML directives, and it also can contain references to properties in the im, xreg, and evt objects. When the mouse hovers over an object, a tooltip string is generated by macro-expanding the values for these properties. The tooltip string is displayed as the inner HTML of the tooltip. When the mouse leaves the object, the tooltip is hidden.

For example, consider the following tooltip string: 

    opts.tooltip = "<b>id: $im.id</b><br>pos: $xreg.x $xreg.y<br><i>$xreg.data.tag</i>";
When the mouse hovers over an object, the displayed tooltip will contain the current image id in bold, followed by that object's position, followed by a "tag" property passed in the data object when the shape was added.
